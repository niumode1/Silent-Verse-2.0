# Silent Verse (无言史诗)

第一人称 3D 硬核生存轮回 MMO。核心驱动不是 PvP，而是一个**由现实物理化学参数完全定义的世界**——玩家砍树、挖矿、冶炼、建造、种地、形成社会。单世界单服，匿名轮回，全玩家生态，零系统中介。

## 设计哲学

1. **现实参数驱动** — 物品行为由其物理/化学属性决定，不存在"木剑伤害=5"的硬编码
2. **物质守恒** — 砍树→木材+木屑，烧木→灰烬+CO₂+热量，总质量不变
3. **零预设配方** — 石+木+绑扎=矛，物理自动推导结果
4. **服务端权威** — 所有物理计算、伤害判定、反应验证在服务端完成，客户端仅表现

## 全局物理常数

### 引力与行星
- **万有引力常数** G = 6.67430×10⁻¹¹ N·m²/kg²
- **行星周长** ~25 km（每赛季由开发者设定），**表面重力** g₀ = 9.81 m/s²
- **重力矢量** g⃗(r⃗) = -G·M·r̂/|r⃗|²，始终指向球心，随海拔衰减 g(h) = g₀·(R/(R+h))²
- **自转周期** 86400 s（=1 游戏天），**自转轴倾角** 23.44°

### 大气
- **海平面** P₀=101325 Pa, T₀=288.15 K, ρ₀=1.225 kg/m³
- **气压高度公式** P(h) = P₀·exp(-h/8500)，空气密度正比于气压
- **空气阻力系数表** Cd（球体 0.47/圆柱 0.82/站立人 1.2/蹲伏 0.8/流线 0.04）

### 水
- ρ_water=1000 kg/m³, 海水 1025, 冰 917
- T_freeze=0°C, T_boil=100°C（随海拔：-1°C/300m）
- η(20°C)=1.002×10⁻³ Pa·s, γ=0.0728 N/m, cp=4184 J/(kg·K), Lv=2260 kJ/kg

### 声/光
- **声速** c_sound = 331.3+0.606·T ≈ 340 m/s (15°C)，随温度变化
- **光速** c_light = 299792458 m/s

### 运动 & 动力学
- 自由落体 v=√(2gh)，终端速度 v_t=√(2mg/ρACd)
- 动能 E_k=½mv²，势能 E_p=mgh，动量 p=mv
- 冲量 J=F·Δt=Δp，向心力 F_c=mv²/r=mω²r

### 摩擦（材料对×表面状态）
- **静摩擦** F_s ≤ μs·N，**动摩擦** F_k = μk·N，**滚动阻力** F_rr = Crr·N
- 表面状态修正：干燥×1.0 / 潮湿×0.7 / 结冰×0.15 / 有油×0.4
- **Capstan 方程** T_load = T_hold·e^(μs·θ)（绑扎牢固度）

### 材料力学
- 应力 σ=F/A，应变 ε=ΔL/L₀，胡克定律 σ=E·ε
- 莫氏硬度判定：H_A > H_B → A 能划伤 B
- 屈服 σ>σy→永久变形，断裂 σ>σt→断裂
- 压强 P=F/A（尖端曲率半径决定接触面积→决定穿透/钝伤）

### 碰撞
- 弹性 v₁′=((m₁-m₂)v₁+2m₂v₂)/(m₁+m₂)，非弹性 ×恢复系数 e
- e 参考：钢~0.9/木~0.6/石~0.3/肉体~0.05

### 简单机械
- **杠杆** F₁·L₁=F₂·L₂，MA=L₁/L₂（三类杠杆均实现）
- **滑轮组** F=mg/n（n=承重绳数）
- **斜面** F=mg·sinθ+μk·mg·cosθ
- **楔子** F_split=F_in/(2·sinα+μk·cosα)
- **轮轴** MA=R_crank/R_drum, 滚动阻力 F_rr=Crr·mg

### 流体
- **浮力** F_buoy=ρ_fluid·V·g（木筏承重判定）
- **液体压强** P=P₀+ρ·g·h（水坝受力、连通器、虹吸）

### 热学
- **传导** dQ/dt=-k·A·ΔT/d，**温升** ΔT=Q/(m·cp)
- **相变** Q_melt=m·Lf, Q_vapor=m·Lv
- **燃烧** Q_out=m·HV·η（η 受氧气供应影响）
- **热膨胀** ΔL=α·L₀·ΔT

### 转动
- 力矩 τ=F·r·sinθ，转动惯量 I=Σmᵢrᵢ²，角动量 L=Iω，转动动能 E_rot=½Iω²

## 核心系统

### 1. 世界系统
真实球形 3D 星球，第一人称视角，地表可环绕，陆地/海洋比 4:6。每赛季世界由开发者用不同种子程序化生成，大小可调整。

### 2. 物理参数体系
每种材料携带 **40+ 个属性**：力学（密度/硬度/屈服强度/抗拉/抗压/剪切/弹性模量/剪切模量/体积模量/泊松比/断裂韧性/疲劳极限/静摩擦/动摩擦/滚动阻力/恢复系数/阻尼/耐磨）、热学（熔点/沸点/比热容/导热/热膨胀/熔化潜热/汽化潜热/闪点/辐射率）、化学（可燃性/反应活性标签/水溶性/毒性/pH/抗氧化/腐蚀性）、流体（粘度/表面张力/休止角）、声学（声速/吸声）、光学（不透明度/反射率/折射率）、结构（最大跨度/砂浆粘结/握钉力）、生态（NPK/可降解/可食性）。物品行为完全由此属性表推导。

### 3. 化学反应系统
五类反应模式覆盖全部玩家操作：**燃烧**（可燃物+O₂+T>闪点→灰烬+CO₂+H₂O+热量）、**碳热还原**（金属氧化物+碳+T>还原温度→纯金属+CO₂+炉渣）、**热分解**（CaCO₃→CaO+CO₂）、**溶解**（草木灰+水→碱水；油脂+碱水→肥皂）、**发酵/腐败**（谷物→酒→醋；肉→腐败）。所有反应质量守恒。

### 4. 赛季系统
1 赛季 = 8 游戏年 = 128 现实天。赛季结束世界完全重置。最后 16 天触发「长冬」——温度骤降、资源不再可用。最终存活者/群体成为赛季「传说」，其事迹作为下一赛季的遗迹文本。

### 5. 资源系统
赛季开局一次性程序化生成全部资源（地形/矿产/植被/土壤 NPK），全程不刷新。树木生长取决于阳光/水/土壤营养，所有物质不消失只转化。

### 6. 时间系统
1 现实天 = 1 游戏天，1 游戏月 = 4 天，1 游戏年 = 16 天。树木生长、发酵、伤口愈合按游戏天计算。物理碰撞/重力不受游戏时间影响。

### 7. 生命系统
人类寿命约 4 游戏年（~64 天），动物 2 游戏年。人类前 3 次投胎免费，后渐进收费（1→3→5→20→50→100→200 元），赛季上限 10 次。动物投胎始终免费，受种群比例管控（自然生态金字塔）。先天属性随机（体格/耐力/智力/视力/代谢 0.7-1.3），低概率先天疾病。锻炼可提升力量和耐力。

### 8. 全玩家生态
所有动物由玩家扮演，无 AI 动物。动物有种群比例硬限制（如 1 狼:3 狐狸:10 兔）。动物有非对称能力（狼强攻击/兔高隐匿+钻洞+感知）。同物种有简单的信号/情感社交。扮演猎物是一种「恐怖生存」体验。

### 9. PvP 系统
开放 PvP，无系统安全区。但 PvP 不是核心——是社会运行中可能出现的行为。伤害由物理公式计算（材料硬度比/压强/动能转移/穿透vs钝伤判定）。

### 10. 离线留存
下线角色原地存在，有简单代谢模拟。有真人玩家靠近时完全激活（类似挂机玩家）。玩家可自建藏身处（降低代谢衰减/隐藏/保暖）、设置陷阱/警报、社区轮班守卫。

### 11. 通信系统
近距离范围文字/语音（WebRTC P2P），远距离靠信使或物理物品记录（刻木/石板）。无全局聊天/公告。

### 12. 建造系统
零预设配方——玩家任意组合材料，服务端按材料属性+几何形状+连接方式（绑/钉/嵌）推导工具属性。绑扎牢固度由 Capstan 方程计算（缠绕圈数×摩擦系数×张力）。

### 13. 冶炼与锻造
完整金属加工链：采矿→选矿→建炉（粘土+水+草→烧陶）→装炉（矿石+木炭+助熔剂）→鼓风冶炼（T>1200°C→海绵铁）→锻打排渣→淬火（奥氏体化→马氏体）→回火调质→磨刃。全过程由热学+化学公式驱动，无一行硬编码。

### 14. 匿名轮回
无固定 ID/角色名，死亡重生=全新身份。外观不可自选，别人无法认出你。死亡=全部物品丢失，但建筑留存为废墟，跨生命信息传递靠物理笔记/刻痕/口述。

## 开发阶段

### Phase 1：原型验证（当前目标）
- **目标** 50 人同时在线，验证核心物理 + 建造 + 冶炼循环
- **架构** 单 Godot Headless 进程，SQLite 持久化
- **服务器** 4 核 8G Linux VPS
- **交付物** 物理计算器（全公式）+ 材料数据库（50种）+ 反应系统 + 基础网络 + 离线实体 + 2人联机验证

### Phase 2：扩展期
- **目标** 200 人，空间分区多 Cell，Redis+PostgreSQL
- **前提** Phase 1 验证通过

### Phase 3：正式运营
- **目标** 数千人，完整 SpatialOS 风格网格
- **前提** Phase 2 稳定运行

## 开发约定

- **物理优先**：所有游戏行为必须先定义物理/化学参数，再写逻辑
- **材料即配方**：新增物品 = 新增材料数据，不改代码
- **服务端权威**：所有计算在服务端完成并验证，客户端仅表现
- **TDD 物理**：物理公式（打击伤害/砍伐/挖矿/反应）必须有单元测试验证
- **质量守恒**：任何反应必须输入质量 = 输出质量
- **单人友好**：代码优先简单可维护，避免过度工程化

## 边界

### Always
- 服务端权威计算，客户端仅表现
- 先定义材料属性，再写玩法逻辑
- 新增反应 → 添加到 reaction_registry，不改引擎代码
- 服务器启动时从 JSON 加载材料数据，运行时只读

### Never
- 硬编码游戏配方 / 配方表
- 硬编码物品属性数值（所有属性从材料+形状推导）
- 信任客户端输入
- 物质凭空产生或消失
